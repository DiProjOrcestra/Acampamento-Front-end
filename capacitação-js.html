🧙‍♂️ JavaScript: A Magia Viva do Reino Web

Dando vida ao Portfólio do Herói

HTML constrói o castelo, CSS o deixa bonito, e JavaScript faz as portas abrirem, as tochas acenderem e os botões reagirem.

📖Prólogo: O Feiticeiro do Reino

JavaScript é a linguagem que permite que a página reaja a cliques, teclas, envios de formulários, animações e qualquer outro evento no navegador.

Nesta jornada, você vai aprender a usar JavaScript para controlar o DOM, lidar com eventos e criar pequenas magias interativas dentro do seu próprio portfolio.html.

Onde JavaScript roda?

  • No navegador (Chrome, Firefox, etc.), manipulando páginas web.
  • No servidor, com Node.js (não é foco aqui, mas será útil em NestJS depois).
Importante: Em todo este livro você vai trabalhar sobre o mesmo portfolio.html estilizado no livro de CSS, adicionando um arquivo main.js ligado a ele.

🔮Nível 1: Invocando o Primeiro Feitiço

Ligando um arquivo JavaScript à página

JavaScript aparece em um arquivo .js, que você conecta no HTML com a tag <script>.

Para o seu portfólio, o objetivo é ligar um main.js ao portfolio.html.

<!-- Dentro de portfolio.html -->
<body>
  ...
  <script src="main.js"></script>
</body>
</html>
// main.js
console.log("Portfólio do Herói: magia JavaScript iniciada!");
alert("Bem-vindo ao Painel do Herói!");

Use o DevTools (F12) e a aba Console para verificar mensagens de log e veja o alert ao abrir a página.

🎯 Missão: Conectando o Portfólio

  1. Na mesma pasta do seu portfolio.html, crie main.js.
  2. Adicione a tag <script src="main.js"></script> antes de </body>.
  3. Escreva um console.log e um alert de teste e confirme no navegador.
🏆 Recompensa: +40 XP – Você ligou JavaScript ao seu portfólio e despertou o primeiro alerta mágico.

⚠️ Armadilhas

  • Nome do arquivo JS diferente do que está em src.
  • Colocar o <script> no <head> sem defer e tentar acessar elementos que ainda não existem.

🧾Nível 2: Atributos do Herói (Variáveis e Tipos)

Variáveis: slots de inventário

Variáveis são “espaços” onde você guarda informações que podem mudar ao longo do jogo. 

let nomeDoHeroi = "Arkan, o Verde"; // pode mudar
const nivelInicial = 1;             // não muda
let pontosDeVida = 100;
let possuiCajado = true;

Use let para valores que mudam e const para valores fixos.

Tipos básicos

const nome   = "Arkan";   // string
let nivel    = 5;          // number
let vivo     = true;       // boolean
let mana;                  // undefined
const nada   = null;       // null

🎯 Missão: Ficha do Herói (dados em JS)

  1. No main.js, crie variáveis com let e const para: nome, classe, nível, vida, mana e ouro.
  2. Use console.log para montar uma frase com esses dados.
  3. Depois use alert para mostrar essa ficha em uma janelinha.
🏆 Recompensa: +50 XP – Seu inventário de variáveis está criado.

⚔️Nível 3: Decisões e Ciclos (if, else, loops)

if/else: escolhas na história

Você usa if e else para tomar decisões com base em condições.

let nivel = 8;

if (nivel >= 10) {
  console.log("Você pode entrar na Masmorra Épica!");
} else {
  console.log("Você ainda é fraco para este desafio...");
}

Laços: repetindo feitiços

for (let i = 1; i <= 3; i++) {
  console.log("Lançando bola de fogo nº " + i);
}

let mana = 3;
while (mana > 0) {
  console.log("Você ainda tem " + mana + " de mana.");
  mana--;
}

🎯 Missão: Cálculo de Nível

  1. Crie uma variável xp e use if/else para decidir o “título” do herói conforme o XP.
  2. Use um for para simular 10 batalhas somando XP.
  3. Ao final, use alert para mostrar o nível e o título final do herói.
🏆 Recompensa: +60 XP – Você controla o fluxo da história.

📜Nível 4: Tomos de Magia (Funções)

Funções: feitiços reutilizáveis

Funções são blocos de código que você pode “invocar” várias vezes, com parâmetros diferentes. 

function saudarHeroi(nome) {
  console.log("Saudações, " + nome + "!");
  alert("Saudações, " + nome + "!");
}

saudarHeroi("Arkan");
saudarHeroi("Luna");

Funções com retorno

function calcularDano(base, forca) {
  const dano = base + forca * 2;
  return dano;
}

const danoFinal = calcularDano(10, 5);
console.log("Dano causado: " + danoFinal);

🎯 Missão: Funções do Painel

  1. Crie uma função subirNivel(xpAtual) que retorna o novo nível com base no XP.
  2. Crie uma função formatarFicha(heroi) que recebe um objeto e devolve uma string com a ficha.
  3. Use alert(formatarFicha(heroi)) para ver a ficha completa.
🏆 Recompensa: +60 XP – Seu grimório agora tem feitiços reutilizáveis.

🎒Nível 5: Inventário e Bestiário (Arrays e Objetos)

Arrays: listas de itens

Arrays são como um inventário com vários slots numerados.

const mochila = ["Poção de Vida", "Poção de Mana", "Chave Enferrujada"];

console.log(mochila[0]); // Poção de Vida
mochila.push("Mapa do Calabouço");

const heroi = {
  nome: "Arkan",
  classe: "Mago",
  nivel: 10,

console.log(heroi.nome);
console.log(heroi["classe"]);

🏰Nível 6: Manipulando o Mundo (DOM e Eventos)

DOM: o mapa do castelo

O DOM é a representação da página que o navegador expõe para o JavaScript como uma árvore de nós.

Você pode buscar elementos, mudar textos, classes, estilos e reagir a eventos como cliques e envios de formulário.

Exemplo: botão que aumenta XP

<!-- Em portfolio.html (dentro do painel-heroi) -->
<p>XP: <span id="hero-xp">0</span></p>
<button id="btn-xp">+10 XP</button>
// Em main.js
let xp = 0;
const btnXp = document.querySelector("#btn-xp");
const spanXp = document.querySelector("#hero-xp");

btnXp.addEventListener("click", function () {
  xp += 10;
  spanXp.textContent = xp;
});

🎯 Missão: Painel do Herói Dinâmico

  1. No portfolio.html, garanta que existam elementos com ids hero-nome, hero-classe, hero-nivel, hero-xp e um botão btn-xp.
  2. No main.js, use document.querySelector para buscar esses elementos.
  3. Ao carregar a página, preencha os spans com os dados do objeto heroi criado no Nível 5.
  4. Ao clicar no botão, aumente o XP e, se passar de certo valor, use if/else para subir o nível e mostrar um alert de evolução.

🌐Nível 7: Magia Assíncrona (async/await e GET)

Chamadas assíncronas: buscar dados no reino

Com fetch e async/await você faz requisições GET para APIs, busca dados externos e atualiza a página sem recarregar. 

// Exemplo simples de GET assíncrono
async function buscarPiada() {
  try {
    const resposta = await fetch("https://api.chucknorris.io/jokes/random");
    const dados = await resposta.json();
    console.log(dados.value);
    alert("Piada mágica: " + dados.value);
  } catch (erro) {
    console.error("Erro ao buscar piada", erro);
    alert("Falha ao invocar a magia remota.");
  }
}

🎯 Missão: Chamado ao Oráculo (GET)

  1. Crie um botão no seu portfolio.html chamado “Consultar Oráculo”.
  2. No main.js, crie uma função async function consultarOraculo() que faz um GET usando fetch.
  3. Mostre o resultado com alert ou inserindo texto em um elemento do DOM.
  4. Use const para a URL da API e let para dados que mudam.
🏆 Recompensa: +60 XP – Você domina a magia assíncrona e está pronto para falar com APIs.

🐉Boss Final: Portfólio do Herói (HTML + CSS + JS)

Mini aplicação: Portfólio Vivo

Projeto final: transformar o seu portfolio.html em uma página interativa completa, com ficha dinâmica no “Painel do Herói”, contador de XP, um formulário de contato com validação simples e pelo menos uma chamada assíncrona GET simulando uma API.

Requisitos mínimos

  • Usar o mesmo portfolio.html e style.css dos livros anteriores (não criar outro HTML).
  • Criar main.js e importá-lo no final do body.
  • No “Painel do Herói”:
    • Preencher automaticamente nome, classe, nível e XP lendo um objeto heroi em JS.
    • Implementar um botão “+XP” que aumenta o XP na tela e sobe o nível ao atingir limiares, usando if/else e alert quando o herói evoluir.
  • No formulário de contato:
    • Impedir envio se nome ou email estiverem vazios.
    • Mostrar uma mensagem de sucesso/erro abaixo do formulário sem recarregar a página.
  • Adicionar uma função assíncrona que faça um GET (API real ou mock) e exiba o resultado.

🎯 Boss Mission (Livro 3)
